# 全局自定义动态背景插件 global-video-bg-plugin
## 功能介绍
1. 完全独立渲染动态视频背景，无需依赖任何其他自定义背景插件，单独安装即可生效
2. 本地视频永久二进制持久化存储，重启客户端/电脑背景配置不丢失
3. 支持上传本地视频作为全局底层动态壁纸，视频限制最大300MB
4. 兼容 mp4 / webm / mov / avi / mkv 五种主流视频格式
5. 0.1~1.0透明度滑块实时调节，一键保存/清除配置
6. 插件互斥机制：检测到其他背景/壁纸插件时自动隐藏本插件背景，弹窗锁定所有操作，杜绝多层画面叠加
7. 保存配置后全页面实时刷新，无需重启客户端
8.配合自定义背景插件，上传静态图片后将背景透明度改为0%，然后保存，即可只有自定义动态背景显示。

### 沉浸式预览更新说明
1. 点击悬浮球进入沉浸式模式：左侧侧边栏、顶部导航、推荐卡片、歌单、搜索框等**所有UI全部隐藏**，仅保留底部播放控制栏，完整展示底层动态视频背景；
2. 再次点击悬浮球即可退出沉浸式，自动恢复客户端全部界面；
3. 悬浮球支持全窗口任意拖拽，拖动不会误触发预览切换；
4. 实现方案：后台service-worker消息广播调试通信 + 运行时DOM样式覆写UI修改，双逻辑结合；
5. 检测到其他背景插件时悬浮球自动销毁，沉浸式功能同步禁用。

### 存储配额溢出Bug修复说明
1. 移除视频二进制Uint8Array持久化存储逻辑，彻底解决 `kQuotaBytes quota exceeded` 存储超限报错；
2. 仅将透明度、悬浮球开关、视频名称标记等极小文本数据存入storage，无容量溢出风险；
3. 页面加载时唤起本地文件选择框，临时读取视频生成页面生命周期内有效的Blob URL；
4. 视频最大300MB限制、5种格式校验、悬浮球拖拽、沉浸式全屏隐藏UI、插件互斥功能全部保留；
5. 切换页面/重启客户端后需重新选择本地视频文件，规避大容量二进制存储限制。

## 最优存储方案更新（解决上架驳回+300MB视频配额报错）
1. 彻底移除IndexedDB（仓库#146禁止，沙盒环境拦截），采用2MB分块分片存储方案，仅使用官方允许的storage权限；
2. 单视频最大支持300MB，分片规避单键容量限制，彻底修复`Resources::kQuotaBytes quota exceeded`存储超限报错；
3. 视频永久本地持久化，重启软件/电脑无需重新上传，页面自动拼接分片完整播放；
4. 所有原有功能完整保留：悬浮球沉浸式预览、全UI隐藏、视频流畅缓冲防回退、插件互斥、300MB大小校验、五种视频格式支持；
5. 清单文件严格遵循MoeKoeMusic-Plugins上架规范，无违规API与权限，可正常通过自动Action校验与人人工审核。

## 视频流畅播放优化更新
1. 全局缓存视频DOM与数据库配置，沉浸式切换、页面刷新不销毁视频，不会中断播放、不会变成静态画面；
2. 开启GPU硬件分层加速渲染，高分辨率视频大幅降低卡顿、掉帧；
3. 增加视频缓冲卡死修复逻辑，缓冲不足、循环结束自动重置播放，杜绝画面定格静止；
4. 延迟释放视频Blob资源，减少重复解码视频带来的加载延迟；
5. 所有原有功能（300MB永久存储、悬浮球拖拽、沉浸式全UI隐藏、插件互斥、透明度调节）完全保留，无改动。

## 超高画质/高帧率/超长视频播放修复更新
1. 新增缓冲节流锁，解决高码率视频缓冲不足时无限回退、无法完整播放的问题；
2. 限制视频最大缓冲时长12秒，定时清理过期缓冲，控制内存占用，支持几十分钟长视频稳定循环；
3. 修改预加载策略为metadata元数据加载，超大分辨率视频不会一次性阻塞主线程；
4. 视频销毁时完整重置解码器释放内存，避免残留缓存导致播放异常；
5. 沉浸式切换仅隐藏UI，全程保留视频播放实例，不会中断、回退视频；
6. 原有全部功能（300MB永久IndexedDB存储、悬浮球拖拽、全UI隐藏沉浸式、插件互斥、透明度调节）完全不变。
